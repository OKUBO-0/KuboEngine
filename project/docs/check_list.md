# ▼ チェックリスト
01. ファイル構成・ドキュメント
- プロジェクト内のフォルダ構成について、1フォルダ内に8個以上のファイルがある場合にサブフォルダに分けているか。
- Visual Studio のフィルタ構成（.vcxproj.filters）がフォルダ構造と一致している。
- "Debug Win32"、"Debug ARM64"など Platform が削除されている。
- 外部ライブラリが external 等の専用フォルダに分離されている。

02. コメント
- クラスコメントに「目的」「責務」が記載されている。
- 関数コメントに「目的」「引数」「戻り値」が記載されている。
- 関数定義内の4行につき1コメント程度、意図を説明する処理コメントが記載されている。
- コメントアウトされた不要コードが残っていない。

03. コーディング基礎
- 各種識別子にスペリングのミスがない。
- 命名規則（キャメル / パスカルなど）が統一されている。
- マジックナンバーが定数または列挙型として定義されている。
- グローバル変数の使用を排除している。
- グローバル関数の使用を排除している。
- new/delete を自前コードで使用していない（unique_ptr / shared_ptr などを利用）。
- 引数に const 参照を使用すべき箇所で適切に使用している。
- 各関数が 40 行以内に収まっている。
- 類似する名前のソースに類似する処理が多数含まれていないか（冗長なコードの有無）。
- JSON / CSV / XML / Script 等の外部ファイルを活用している。
- エンジン層のコードが namespace で分類されている（外部ライブラリは対象外）。
- モダンC++を活用している。
- 使用しなくなった変数の残骸が残っていない

04. クラス設計
- カプセル化（public / protected / private の分離）が守られている。
- 各クラス定義の行数が 300 行以下に収まっている。
- 継承と仮想関数によるポリモーフィズムが複数個所で利用されている。
- 状態遷移に State パターンが利用されている。
- デザインパターン（Factory / Strategy / Observer など）が、複数種類利用されている。
